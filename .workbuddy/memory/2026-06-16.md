## Skills 用户故事评估

### 评估的 4 个用户角色
1. 新项目开发者 - 需要快速生成标准规范
2. 已有代码库维护者 - 需要为老项目生成规范
3. 团队 Tech Lead - 需要统一管理多项目规范
4. CI/CD 工程师 - 需要集成到 CI 流程

### 发现的主要问题
1. 入口点不够明显，用户需要知道 Skill 名称
2. 缺少 CI 集成，生成的系统不能直接用于 CI
3. 缺少可视化展示，难以理解扫描结果
4. 缺少批量操作，多项目管理效率低

### 优化优先级
- P0: 意图识别、CI 集成、快速开始、进度反馈
- P1: 可视化展示、批量操作、错误分类、FAQ
- P2: 协作功能、报告功能、版本控制

### 与 SDD 工具对比
| 工具 | 定位 | 核心输出 |
|------|------|----------|
| Spec Kit | 规格驱动 | Constitution + Specs + Tasks |
| OpenSpec | 轻量 SDD | specs/ + AGENTS.md |
| Agent Skills | 技能框架 | SKILL.md + 资源 |
| Superpowers | 强制流程 | 7 阶段工作流 |
| Harness Skills | 项目驱动 | 脚本 + Hook + CI + Skill |

### Harness 优势
- 项目感知，自动适配
- 生成即用，CI 集成
- 自然语言，低门槛

### Harness 劣势
- 生态小，社区有限
- 无完整规格生命周期
- 无强制 TDD 流程
- 仅 WorkBuddy 平台
